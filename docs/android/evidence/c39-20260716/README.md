# C39 UI-004 Current iOS Matched Caregiver Login

## Baseline and capture method

- Product baseline: `main@1cf8aef`; Android starting point: `android-dev@3622bd4`.
- Current iOS source was built and launched without source edits on a disposable iPhone 17e / iOS 26.5 simulator. The checked-in UI-test caregiver bootstrap and login callback route were used.
- The filled references use synthetic `care@example.com` / `SamplePass123!` values for display only. No login action or credential network request was performed.
- Android captures use the production `CaregiverAuthFlow` on the API 35 AVD. Light, dark and 200%-font fixtures are generated by instrumentation tests.

## Drift closed

- Isolated the login/signup form treatment from the teal entry/auth-choice palette and matched the current iOS light/dark system-blue accent.
- Replaced the app-tinted screen background with the current white/black form canvas.
- Reproduced the 24-unit glass-card hierarchy with semantic light/dark surfaces, subtle stroke and elevation.
- Replaced outlined inputs with rounded quaternary-fill equivalents and preserved Email Next -> Password / Password Done behavior.
- Matched the SwiftUI outer 28-unit form spacing, 12-unit field stack, 50-unit blue primary action and blue-at-50%-opacity disabled action.
- Replaced the long Material back arrow with the circular chevron navigation treatment already established by the current iOS flow.

SF Symbols and Material glyph outlines, keyboard artwork and system bars remain platform-native. Copy, ordering, card geometry, semantic colors, enabled/disabled hierarchy and navigation meaning match the current runtime.

## Automated result

- `CaregiverAuthFlowScreenTest`: 16/16 passed on API 35.
- The matched fixtures assert empty, synthetic-filled light/dark and dark 200%-font states without submitting the synthetic credentials.
- Existing login/signup IME, loading, invalid-credential, validation, confirmation and resend lifecycle regressions remain in the same suite.
- Real credentials, physical keyboard/OEM IME and TalkBack traversal remain Gate I evidence.
