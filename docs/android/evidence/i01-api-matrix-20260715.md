## Findings and corrections

The first cross-API run exposed test infrastructure assumptions rather than production failures:

- `FileInputStream.readNBytes` was unavailable on API 26; the PDF signature assertion now uses the minSdk-compatible `read(byte[])` path.
- A screenshot fixture wrote through scoped-storage MediaStore APIs and required legacy external-storage permission on API 26; fixtures now stay in private app cache and do not pollute user media.
- Several LazyColumn assertions assumed API 35 viewport composition and counted off-screen duplicate nodes. They now scroll through the production container or assert semantics/actions independently of viewport height.

The expanded adaptive suite then exposed a real compact-viewport issue in the PDF custom-range sheet: its validation and generate action could fall below the non-scrollable API-26 viewport. The production sheet is now vertically scrollable and the regression test scrolls to both targets.

After the corrections, the focused classes passed on all three APIs, followed by the complete matrix on the final code. The 2026-07-15 XP-007 extension added five production-Compose stale/retry scenarios. C06 then added four 200%-font Caregiver reachability scenarios for Today, medication save, inventory correction and History day detail. The expanded suite passes 93/93 on API 26, 33 and 35 (279/279 total). No production UI/API behavior was weakened to satisfy the matrix.
